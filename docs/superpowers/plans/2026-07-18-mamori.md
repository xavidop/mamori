# mamori Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `mamori` - a Go library that loads config & secrets from heterogeneous sources into typed, validated structs and keeps them reconciled at runtime - plus a provider ecosystem, middleware, conformance kit, `go vet` analyzer, OTel bridge, an Astro docs site, and GoReleaser release pipeline.

**Architecture:** Multi-module monorepo. A dependency-light **core** module (`github.com/xavidop/mamori`) owns the generics API (`Load[T]`/`Watch[T]`), the Ref grammar, the provider SPI, the reconciler engine, secret types, built-in providers (env/file/exec), middleware, and the conformance kit. Each cloud provider is its **own module** under `providers/*` so SDK deps never reach core consumers. `x/otel` and `tools/reconcilevet` are separate modules. `go.work` unifies local dev.

**Tech Stack:** Go 1.26; `go-playground/validator/v10`; `go-viper/mapstructure/v2`; `fsnotify/fsnotify`; `goleak` (tests); provider SDKs (aws-sdk-go-v2, cloud.google.com/go, azsecrets, vault/api, client-go, consul/api); `golang.org/x/tools/go/analysis` (vet); Astro (docs); GoReleaser + GitHub Actions.

## Global Constraints

- Module root is `github.com/xavidop/mamori`. Root package name is `mamori`.
- **Core dependencies are limited to:** stdlib, `github.com/go-playground/validator/v10`, `github.com/go-viper/mapstructure/v2`, `github.com/fsnotify/fsnotify`, and `github.com/uber-go/goleak` (test-only). NO cloud SDK, NO OTel, NO clockwork in core.
- Every exported symbol has a doc comment. Public API surface matches the spec §7.
- All time in the engine flows through the `mamori.Clock` interface - never call `time.Now()`/`time.After` directly in engine code, so tests are deterministic.
- Secret values must never reach `fmt`/logs unredacted.
- TDD: write the failing test first, watch it fail, implement minimally, watch it pass, commit. Small commits.
- Each provider module’s `go.mod` uses `go 1.26` and a `replace github.com/xavidop/mamori => ../..` for local dev (also covered by `go.work`); the published module drops the replace via release tooling.
- Every provider that can, passes `providertest.Run` against an in-memory fake backend. Live-backend integration tests are `//go:build integration` and are NOT run in this build.
- Commit trailer on every commit: `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.

**Produces:**

- `func Watch[T any](ctx context.Context, opts ...Option) (*Watcher[T], error)` - initial `Load` (fail-fast); start per-ref watch (native `WatchableProvider.Watch` if implemented, else `pollWatch`); start reconciler goroutine.
- `type Change[T any] struct { Old, New T; Fields []FieldChange }` with `func (c Change[T]) Changed(path string) bool`.

- `func OnChange[T any](fn func(Change[T])) Option` and `func OnError(fn func(error)) Option`.
- [ ] **Test first**: `Watch` returns after resolving initial config; `Get()` returns it; no `OnChange` on startup.
- [ ] **Test**: a fake watchable provider pushes a new Version → after debounce, `OnChange` fires with `New` reflecting the change and `Fields` naming the path.
- [ ] **Test (atomicity)**: pushed value fails `validate` → `OnChange` does NOT fire, `Get()` unchanged, `OnError(ValidationError)` fires.
- [ ] Run each → FAIL then PASS incrementally. Commit `feat: Watch engine - atomic revalidated updates`.

### Task 10: coalescing, ordering, backoff, stale, shutdown
**Files:** Modify `reconciler.go`; Create `watch_semantics_test.go`.
**Consumes:** Task 9.
**Produces:** debounce/coalesce window (default 500ms, per-field `?debounce=`); single serialized dispatch goroutine with bounded drop-oldest queue (`WithQueueDepth`, default 16); per-ref exponential backoff w/ cap on resolve error (retain last-good, `OnError(ProviderError)`); `WithStale(maxAge)` escalation; `Close()` cancels watches, drains queue, returns.
- [ ] **Test (coalescing)**: five field Updates within the window → exactly one `Change` with five `Fields` (fake clock drives the window).
- [ ] **Test (ordering)**: a slow `OnChange` delays but does not drop the next event; overflowing the queue drops oldest (assert last delivered).
- [ ] **Test (backoff)**: provider returns error → `Get()` keeps last-good, `OnError(ProviderError)` fires, retry interval grows (fake clock).
